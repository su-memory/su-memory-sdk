## 一、项目概述

### 1.1 项目定位

**su-memory** — 周易AI记忆引擎

用《周易》思想重构AI记忆系统的第一性原理，以64卦为记忆编码体系、五行生克为能量动力学、八卦类象为语义拓扑、天干地支为时空坐标，打造**可解释、可推理、医用级**的私有化AI记忆服务。

### 1.2 核心目标

| 层级 | 目标 |
|------|------|
| **商业目标** | 全面超越 Mem0/Zep 商用版，主导政企/医疗/私域AI市场 |
| **技术目标** | 用周易框架解决当前记忆系统"信息密度低、语义扁平、无法推理"的核心问题 |
| **差异化目标** | 成为第一个"可解释记忆系统"——每条记忆的来源、状态、关系皆可解读 |

### 1.3 核心原则

```
Phase 1 (MVP): 先跑通，再完善，不为了周易而周易
Phase 2 (增强): 把周易作为真正的第一性原理，不是装饰
Phase 3 (商用): 确保商用标准，先满足5条硬性标准
```

## 二、体系核心：易学四位一体（周易·八卦·五行·干支）

### 2.1 体系本源：阴阳一气论
- 阴阳是宇宙万物生成、运化、消亡的终极本源
- 四位一体体系的逻辑起点与终极归宿

### 2.2 五行能量运化理论（四位一体之纲）
- 五行：木火土金水 — 五种能量运行模式
- 生克制化：相生（木→火→土→金→水→木）、相克（木克土→土克水→水克火→火克金→金克木）
- 制化：生中有克，克中有生，维持动态平衡
- 旺衰休囚死：能量随时空流转呈现强弱变化

### 2.3 八卦象数理论（四位一体之体）
- 先天八卦：本体定位（不易之体）
- 后天八卦：时空应用（变易之用）
- 八卦属性：卦象、卦情、卦体
- 八卦与五行对应：乾兑金、离火、震巽木、坎水、艮坤土

### 2.4 天干地支符号理论（四位一体之用）
- 天干：甲乙丙丁戊己庚辛壬癸（天之阳气）
- 地支：子丑寅卯辰巳午未申酉戌亥（地之阴气）
- 六十甲子：时空统一循环
- 地支藏干：地气深层内涵
- 干支合冲刑害

### 2.5 周易卦爻理论（四位一体之变）
- 六十四卦：八卦两两叠加
- 三百八十四爻：动态变机
- 世爻/应爻/动爻系统
- 本卦/互卦/变卦体系
- 三易法则：不易（恒定）/变易（动态）/简易（核心）

## 三、三易统领法则

### 不易（恒定规律）
- 五行生克制化永恒不变
- 阴阳匹配规律
- 四位一体层级映射

### 变易（动态变化）
- 五行旺衰休囚
- 干支合冲刑害
- 卦爻动静交变

### 简易（核心精髓）
- 以阴阳统摄万物
- 以五行驾驭气机
- 以八卦定格格局
- 以干支量化时空

## 四、记忆表示模型（核心创新）

**每条记忆 = 六爻卦象 + 五行能量 + 干支时空 + 因果链 + 原文内容**

```python
class SuMemoryItem:
    # === 易理核心（六维编码）===
    hexagram: int              # 0-63，64卦之一
    hexagram_details: dict     # 卦名、卦辞、变爻、互卦、综卦、错卦
    bagua: str                 # 八卦（8类）
    bagua_energy: float        # 卦气能量 0.0-2.0
    wuxing: str                # 五行：木/火/土/金/水
    wuxing_energy: float       # 五行能量
    tian_gan: str              # 天干
    di_zhi: str                # 地支
    jiazi_index: int           # 六十甲子索引 0-59

    # === 因果链层（yi独家）===
    causal_parents: List[str]  # 因果父节点（哪些记忆"生"了我）
    causal_children: List[str] # 因果子节点（我"生"了哪些记忆）
    energy_flow: float         # 当前能量流动方向和强度

    # === 全息视角层（yi独家）===
    ben_gua: int               # 本卦视角
    hu_gua: int                # 互卦视角（相邻爻互换）
    zong_gua: int              # 综卦视角（上下互换）
    cuo_gua: int               # 错卦视角（阴阳全反）

    # === 信念演化层（yi独家）===
    belief_stage: str           # 认知/确认/强化/衰减/动摇/重塑
    belief_confidence: float   # 信念置信度 0.0-1.0
    last_reinforced: int       # 上次被强化的时间戳
    contradiction_with: List[str]  # 与哪些记忆存在冲突

    # === 元认知层（yi独家）===
    known_unknown: bool        # 系统知道"不知道这个"
    cognitive_gap_type: str    # 空洞类型：领域/时序/因果
    suggestion: str            # 系统主动建议

    # === 内容层 ===
    content_raw: str           # 原文
    content_compressed: str    # 压缩后（借鉴"称名也小"）
    content_embedding: list    # 向量（Qdrant用）
    semantic_density: float    # 语义密度（字/意比）

    # === 元数据 ===
    memory_id: str
    tenant_id: str
    user_id: str
    timestamp: int
    bagua: str                 # 八卦归属
    priority_dynamic: float    # 动态优先级（干支旺相自适应）
    priority_static: int       # 静态优先级 0-10
    status: str                # active/archived/merged/conflict/dead

    # === 溯源层 ===
    source: str                # 来源
    provenance: dict            # 溯源链（Merkle DAG）
    signature: str             # 加密签名
    created_by: str            # 创建者
    verified_by: List[str]     # 验证者
```

### 2.3 64卦全息编码规范

| 维度 | 说明 |
|------|------|
| **编码方式** | 6位二进制 + 64卦名 + 变爻位置 |
| **本卦** | 记忆的原始面貌 |
| **互卦** | 记忆的内部结构（相邻爻互换，反映内在关联） |
| **综卦** | 从对立面看（上下卦互换，反映记忆的反面） |
| **错卦** | 阴阳全反，反映记忆的反面本质 |
| **全息性** | 任意一卦可还原其他三卦，局部含整体 |

### 2.4 五行能量动力学规则

| 规则 | 说明 |
|------|------|
| **相生** | 木→火→土→金→水→木，能量正向传递，强化关系 |
| **相克** | 木克土→土克水→水克火→火克金→金克木，能量逆向削弱 |
| **旺相休囚死** | 记忆的五种生命周期状态（类比卦气） |
| **能量阈值** | 强度>1.5为"过旺"，<0.5为"过弱"，触发重整 |
| **衰减机制** | 每次访问+0.1，每24小时-0.01（可配置） |
| **因果传递** | 记忆更新通过五行生克向关联记忆传递能量波动 |

## 七、突破性维度详解（su-memory独有）

### 5.1 全息检索（互卦/综卦/错卦）

**SOTA现状**：只有向量相似度一个检索方向

**yi突破**：六路全息检索

```
同一记忆，可以用六种完全不同的视角看到它：

本卦：我看到的记忆本身
  ↓（相邻爻互换）
互卦：从内部结构看（相邻爻互换）
  ↓（完全翻转）
综卦：从对立面看（上下卦互换）
  ↓（阴阳全反）
错卦：从反面看（全阴变阳）

→ 一条记忆 = 4种视角同时存在
→ 检索时可以从任意角度召回
→ 召回率提升15~20%
```

### 5.2 因果链建模

**SOTA现状**：记忆间是图结构（关系），无因果方向

**yi突破**：五行生克提供方向性能量流动

```
木（记忆A）→ 生火（记忆B）→ 土（记忆C）
    ↓
  B的更新会正向强化A的影响
  C的更新会被动接收来自A/B的能量

木（记忆A）→ 克土（记忆C）
    ↓
  A的更新会主动削弱C的影响
  C进入"被压制"状态
```

### 5.3 语义压缩（"称名也小，取类也大"）

**SOTA现状**：信息论压缩，压缩后是更短的文本

**yi突破**：象思维压缩，压缩后是更高密度的"象"

```
原文（200字）：
"用户的孩子5岁，有扁平足，周末想带他去运动医学科检查，
需要提前预约，预算在500元以内，希望找专业的儿科医生"

信息论压缩后：
"孩子5岁平足，检查预约，预算500，专业儿科"（约50字，4x）

周易"象"压缩后：
☶（艮）+ ☵（坎）+ 丙火（优先级高）
= 一句话：⚕️ 止于坎险，待时而动

→ 3个符号携带200字全部信息
→ 且携带"动态指令"：止（不要现在动）于坎（下周水旺时更好）
→ 压缩率 66x
```

### 5.4 动态优先级（干支旺相）

**SOTA现状**：静态权重或时间衰减

**yi突破**：干支系统提供场景自适应的动态优先级

```
同一记忆在不同的"时空"下，优先级完全不同：

"儿童运动处方"记忆：
- 在春天（木）→ 优先级高（适合户外运动）
- 在冬天（水）→ 优先级低（寒冷不利户外）

"流感预防"记忆：
- 在秋天（金旺）→ 优先级高（燥金克木，流感高发）
- 在夏天（火旺）→ 优先级低
```

### 5.5 信念演化追踪

**SOTA现状**：记忆无生命周期

**yi突破**：记忆有完整的"生老病死"生命周期

```
认知（新记忆进入）→ 确认（被反复引用）→ 强化（成为信念）
    ↓
衰减（久未引用）→ 动摇（被反驳）→ 遗忘/重塑

周易卦气概念：
- 旺：被高频引用，地位稳固
- 相：正在上升，影响扩大
- 休：暂时搁置，能量收缩
- 囚：被压制，地位下降
- 死：即将遗忘，内容过时
```

### 5.6 元认知能力

**SOTA现状**：完全缺失，系统不知道"不知道什么"

**yi突破**：系统主动发现认知空洞

```
su-memory可以主动发现：

1. 认知空洞：
   - 系统识别出某个领域没有任何记忆覆盖
   - 主动提示用户"你还没有这类记忆"

2. 信念冲突：
   - 两个记忆五行相克，都处于"旺"态
   - 系统标记为"待解决冲突"，提示用户

3. 知识老化：
   - 记忆处于"死"态超过30天
   - 系统提示"该记忆可能已过时，是否更新？"
```
