## 二、核心战略：四层闭环

对标 DeepMind 提出的 AGI 三大瓶颈，su-memory 已解决「长效记忆」「高级推理」两大瓶颈，第三步是补齐「持续学习」──通过四层叠加模块形成自主进化闭环：

```
          ┌──────────────────────────────────────┐
          │          四层持续学习闭环               │
          │                                      │
          │  Layer 1 ──→ Layer 2 ──→ Layer 3     │
          │  感知层       归纳层       适配层       │
          │  (增量入库)   (知识蒸馏)   (无遗忘适配)   │
          │     │                        │       │
          │     └──── Layer 4 ◄──────────┘       │
          │            进化层                      │
          │          (自我复盘迭代)                │
          └──────────────────────────────────────┘
```

### Layer 1: 记忆增量入库 (感知层)

**目标**: 从交互/任务/反馈中自动抽取新知识，归档至记忆图谱

**已有基础**: `_incremental_learning.py` 935行 (部分实现), `add()`, `query()`, `recall_trigger.py`, `recency_feedback.py`

**待建设**:
1. LLM推断入口升级 (修复BUG-19 → 关键词30% → LLM 85%+)
2. 交互流自动提取 (对话→记忆, 任务→经验, 反馈→修正)
3. 知识碎片识别与归档 (自动判断哪些信息值得长期存储)

### Layer 2: 知识蒸馏与泛化 (归纳层)

**目标**: 从碎片记忆中提炼通用推理规则与解决范式

**已有基础**: `summarize()` 单主题压缩, `conflict_resolution()` 矛盾检测

**待建设**:
1. 跨记忆模式发现 (Pattern Mining over memory graph)
2. 规则抽象引擎 (碎片→规则, 如 "3次相似诊断→通用营养方案模板")
3. 范式库 (Paradigm Library) - 存储泛化后的可复用知识单元

### Layer 3: 轻量化无遗忘增量适配 (适配层)

**目标**: 不修改基础模型权重，通过记忆路由+规则调用实现新知识落地

**已有基础**: `AdaptiveEngine`, `RetrievalWeightAdapter`, `EncodingDimensionAdapter`, `MemoryForgetting`

**待建设**:
1. 记忆路由表 (Memory Router) - 新知识到达时自动路由到正确的记忆分区
2. 规则触发引擎 (Rule Trigger) - 满足条件时自动激活泛化规则
3. 灾难性遗忘防护 (Catastrophic Forgetting Shield) - 新知识不覆盖旧知识

### Layer 4: 自我复盘迭代 (进化层)

**目标**: 周期性优化记忆权重、检索策略与推理链路

**已有基础**: `PredictionModule`, `ExplainabilityModule`, `FeedbackLoop`

**待建设**:
1. 复盘智能体 (Reflection Agent) - 周期性审视记忆质量
2. 权重自优化 (Auto Weight Optimization) - 基于实际查询效果调整检索权重
3. 进化闭环 (Evolution Pipeline) - "交互→记忆→归纳→优化→升级"

## 三、版本路线图

### v2.1.0: 感知层 - LLM推断激活 (Sprint 1-2)

> **主题**: 激活7000行能量引擎的LLM入口，打通感知层

| 任务 | 内容 | 优先级 |
|------|------|:---:|

| 推断缓存层 | 对相同/相似内容缓存推断结果，避免重复LLM调用 | 🟡 P1 |
| 推断置信度阈值 | confidence<阈值 时回退到能量总线默认值 | 🟡 P1 |

**Sprint 2: 能量链串联 (W3-W4)**

| 任务 | 内容 | 优先级 |

## 六、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|------|
| LLM推断成本高 (API调用) | Layer 1成本爆炸 | 推断缓存 + 批量推断 + 小模型优先 |
| 四层叠加后性能退化 | 查询变慢 | 分层异步执行, 核心路径保持同步 |
| 知识蒸馏产生幻觉规则 | 引入错误知识 | 规则验证器 + 多源交叉验证 + 人工审核接口 |
| 基准退化 (回归) | 核心卖点丢失 | Sprint 12全量回归, 每个Sprint结束跑快速回归 |
| 复盘智能体误删核心记忆 | 知识丢失 | MemoryAnchor保护锁 + 软删除 + 回滚机制 |

## 九、结论

su-memory 项目已完成 AGI 三大瓶颈中的前两个（长效记忆 + 高级推理），且在三榜基准评测中取得 #1 成绩。第三大瓶颈「持续学习」是 v2.5 的核心目标。

通过叠加四层模块（感知→归纳→适配→进化），无需重构现有架构，即可形成完整的 AGI 级认知闭环。这与 DeepMind 提出的 2030 年 AGI 攻坚方向高度吻合——从认知架构突破，而非算力堆砌。

v2.5.0 发布后，su-memory 将从"记忆系统"升级为"AGI 认知底层基础设施"。
