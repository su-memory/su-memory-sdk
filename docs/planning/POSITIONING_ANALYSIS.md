# su-memory 定位分析：医疗专用 vs 通用记忆引擎（第一性原理）

> 日期：2026-07-17
> 依据：项目能力清单审查 + 通用记忆引擎行业调研（Mem0/Zep/Letta/A-MEM/Cognee/LangMem）
> 结论先行：su-memory 在「医疗专用层」全面超越通用引擎，但在「召回质量/安全门控」有红线缺口。

## 一、能力对照矩阵（精简版）

### 通用机械能力（持平第一梯队）
- 向量语义检索、多跳推理、时间感知、记忆演进、持久化并发 —— su-memory 不落后
- Zep/Graphiti 的双时间知识图谱在纯技术深度上更强，差异化不在这一层

### 医疗专用层（6/6 超越，行业唯一）
- 原生 PHI 合规（通用引擎反而扩大攻击面，MRMMIA 论文证明）
- 患者级强制隔离（非软命名空间）
- 内置医疗领域知识（非用户喂本体）
- 跨次就诊纵向连续性
- 反馈训练飞轮 + 临床模式提炼
- FHIR R4 原生互操作

### 红线缺口（6 个，需补齐）
| 缺口 | 性质 |
|---|---|
| C1 医疗同义召回 | 召回质量 |
| C2 记忆抽取层 | 质量 |
| C3 风险门控检索 | 安全关键 |
| C4 双时间模型 | 时序准确 |
| C5 来源溯源 | 合规审计 |
| C6 版本化冲突消解 | 诊疗回溯 |

## 二、定位修正

- 「医疗专用 specialized」= 功能覆盖 ✅ 已达成
- 「医疗级 medical-grade」= 安全保证等级 ⚠️ 需补 C1/C3/C5
