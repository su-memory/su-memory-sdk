## 二、快速部署（30分钟）

### 前置条件

- Linux x86_64 服务器
- Docker 20.10+
- 8GB+ 内存
- 50GB+ 存储

### 步骤1：加载镜像

```bash
# 在目标服务器执行
docker load < su-memory-1.0.0.tar
```

### 步骤2：配置环境

```bash
cp .env.example .env
vim .env  # 修改以下配置
```

必须修改的配置：

```bash
# 数据库密码（必须修改）
POSTGRES_PASSWORD=your_secure_password_here

# JWT密钥（必须修改）
JWT_SECRET_KEY=generate_a_random_64_char_string

# LLM配置（根据实际部署调整）
LLM_BASE_URL=http://your-llm-server:11434/v1
LLM_MODEL=qwen2.5:7b
```

### 步骤3：启动服务

```bash
# 方式一：使用部署脚本
bash deploy/start.sh

# 方式二：手动docker-compose
docker-compose up -d
```

### 步骤4：验证部署

```bash
# 健康检查
curl http://localhost:8000/health

# 预期输出：
{"status": "healthy", "service": "su-memory", "version": "1.0.0"}

# API文档
open http://localhost:8000/docs
```

## 四、集成到你的应用

### 方式一：直接API调用

```bash
# 写入记忆
curl -X POST http://localhost:8000/v1/memory/add \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_001",
    "content": "用户有高血压病史10年",
    "metadata": {"type": "fact"}
  }'

# 检索记忆
curl -X POST http://localhost:8000/v1/memory/query \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_001",
    "query": "用户有什么慢性病",
    "limit": 5
  }'

# 带记忆对话
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen",
    "messages": [{"role": "user", "content": "这个患者该怎么用药"}],
    "user_id": "user_001"
  }'
```

### 方式二：Python SDK

```python
import requests

API_URL = "http://your-server:8000/v1"
API_KEY = "your-api-key"
HEADERS = {"Authorization": f"Bearer {API_KEY}"}

# 写入记忆
requests.post(f"{API_URL}/memory/add",
    headers=HEADERS,
    json={"user_id": "user_001", "content": "用户有高血压"})

# 检索记忆
r = requests.post(f"{API_URL}/memory/query",
    headers=HEADERS,
    json={"user_id": "user_001", "query": "用户健康状况"})
memories = r.json()["memories"]
```

## 五、运维指南

### 5.1 日志查看

```bash
# 应用日志
docker-compose logs -f su-memory

# 向量库日志
docker-compose logs -f qdrant

# 数据库日志
docker-compose logs -f postgres
```

### 5.2 常用运维命令

```bash
# 查看服务状态
docker-compose ps

# 重启服务
docker-compose restart su-memory

# 查看内存使用
docker stats

# 备份数据
docker-compose exec postgres pg_dump -U sumemory sumemory > backup_$(date +%Y%m%d).sql
```

## 六、故障排查

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 启动失败 | 端口占用 | `lsof -i :8000` 查看端口 |
| 连接超时 | 防火墙 | `firewall-cmd --add-port=8000/tcp` |
| 向量检索慢 | 内存不足 | 增加服务器内存 |
| 记忆丢失 | 数据库未持久化 | 检查docker volumes |
| API 401 | API_KEY错误 | 检查请求头格式 |
