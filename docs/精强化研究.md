## 维度一：多跳推理（最大差距：-41%）

### Hindsight 的机制

Hindsight 的多跳推理能力体现在 `CausalInference.multi_hop_inference()` 中：

```python
# su_core/_sys/causal.py 第 200-260 行
def multi_hop_inference(self, query_bagua, query_wuxing, memories, max_hops=3):
    hop_decay = 0.7  # 每跳衰减系数
    
    # First hop: 直接关联
    for i, (m, attr) in enumerate(zip(memories, mem_attrs)):
        rel = self.infer_relation(query, attr)
        if rel["score"] > 0:
            first_hop_results.append((i, score, rel))
    
    # Second hop: 通过 bridge 节点连接
    bridges = first_hop_results[:5]
    for bridge_idx, bridge_score, bridge_rel in bridges:
        for j, attr in enumerate(mem_attrs):
            rel2 = self.infer_relation(bridge_attr, attr)
            hop2_score = bridge_score * hop_decay * rel2["score"]
            # 更新 best_scores[j]
    
    # Third hop: 继续扩展
```

**问题分析**：
1. **跳数限制僵硬** — 最多 3 跳，无法动态调整
2. **Bridge 选择策略简单** — 只取 top-5，没有语义多样性考虑
3. **没有循环图处理** — 遇到环路会重复计算
4. **衰减系数固定** — 0.7 是经验值，没有根据记忆类型调整

### su-memory 的现有能力

su-memory 的 `CausalChain` 提供了多层因果追踪：

- Layer 1: 直接因果（`link()` 方法）
- Layer 2: 八卦语义因果（`link_with_bagua()` 方法）
- Layer 3: 五行能量因果（`link_with_wuxing()` 方法）
- Layer 4: 干支时空因果（`link_with_ganzhi()` 方法）
- Layer 5: 六爻变易因果（`propagate()` 方法）

**关键差距**：
1. su-memory 的 `CausalChain` 是**静态图结构**，需要预先建立连接
2. Hindsight 的 `CausalInference` 是**检索时推理**，无需预建图
3. su-memory 缺少**动态跳数扩展**逻辑

### 解决方案

**新增文件**: `su_core/_sys/multi_hop.py`

```python
class MultiHopRetriever:
    """
    多跳推理检索器
    
    在 CausalInference 基础上增加：
    1. 动态跳数（根据查询复杂度自动调整 1-4 跳）
    2. 多样性 bridge 选择（避免聚集在同一个语义域）
    3. 环路检测与剪枝
    4. 自适应衰减系数（根据记忆类型调整）
    """
    
    HOP_DECAY_BY_TYPE = {
        "decision": 0.75,   # 决策类记忆衰减慢
        "fact": 0.65,        # 事实类记忆正常衰减
        "insight": 0.70,     # 洞察类记忆中等衰减
        "task": 0.60,       # 任务类记忆衰减快
    }
    
    def __init__(self, causal_inference: CausalInference):
        self.inference = causal_inference
        self._max_hops_default = 3
        self._max_hops_complex = 4
    
    def retrieve(self, query_content, query_hexagram, candidates, 
                 max_hops=None, query_complexity="normal") -> List[Dict]:
        actual_hops = max_hops or (
            self._max_hops_complex if query_complexity == "complex"
            else self._max_hops_default
        )
        
        query_bagua = self._get_bagua_name(query_hexagram)
        query_wuxing = self._get_dominant_wuxing(query_hexagram)
        mem_attrs = self._build_mem_attrs(candidates)
        
        if actual_hops >= 2:
            results = self.inference.multi_hop_inference(
                query_bagua, query_wuxing, candidates, max_hops=actual_hops
            )
        else:
            results = self._direct_match(candidates, query_hexagram)
        
        # 多样性重排（避免结果聚集在同一卦象）
        results = self._diversity_rerank(results, candidates)
        return results
    
    def _diversity_rerank(self, results, candidates) -> List[Dict]:
        """MMR (Maximum Marginal Relevance) 变体"""
        reranked = []
        selected_hexagrams = set()
        
        for item in results:
            hex_idx = item.get("hexagram_index", 0)
            novelty = 0.0 if hex_idx in selected_hexagrams else 1.0
            score_semantic = item.get("hop_score", 0)
            score_final = 0.7 * score_semantic + 0.3 * novelty
            
            item["final_score"] = score_final
            reranked.append(item)
            selected_hexagrams.add(hex_idx)
        
        reranked.sort(key=lambda x: x["final_score"], reverse=True)
        return reranked
```

```python
from dataclasses import dataclass
from typing import List, Dict, Optional
import re

@dataclass
class IntentConfig:
    name: str
    keywords: List[str]
    level: int  # 0=闲聊, 1=简单, 2=标准, 3=深度
    wikis: List[str]
    dreams_boost: float
    tags: List[str]

class IntentClassifier:
    """意图分类器 — 基于关键词匹配 + 正则扩展"""
    
    DEFAULT_INTENTS = {
        "project-status": IntentConfig(
            name="project-status",
            keywords=["项目状态", "进度", "任务", "当前在做"],
            level=2, wikis=["obsidian"],
            dreams_boost=0.3,
            tags=["nutri-brain", "task", "progress"]
        ),
        "financing": IntentConfig(
            name="financing",
            keywords=["融资", "路演", "投资", "BP"],
            level=2, wikis=["obsidian", "memex"],
            dreams_boost=0.2,
            tags=["nutri-brain", "financing"]
        ),
        "technical": IntentConfig(
            name="technical",
            keywords=["技术", "架构", "代码", "开发"],
            level=2, wikis=["obsidian", "memex"],
            dreams_boost=0.2,
            tags=["nutri-brain", "technical"]
        ),
        "casual": IntentConfig(
            name="casual",
            keywords=["你好", "天气", "闲聊"],
            level=0, wikis=[], dreams_boost=0.0, tags=[]
        )
    }
    
    def __init__(self, custom_intents: Optional[Dict] = None):
        self._intents = {**self.DEFAULT_INTENTS, **(custom_intents or {})}
        self._keyword_patterns = self._compile_patterns()
    
    def _compile_patterns(self) -> Dict[str, re.Pattern]:
        patterns = {}
        for name, config in self._intents.items():
            pattern_str = "|".join(re.escape(kw) for kw in config.keywords)
            patterns[name] = re.compile(pattern_str, re.IGNORECASE)
        return patterns
    
    def classify(self, query: str) -> IntentConfig:
        query_lower = query.lower()
        for name, config in self._intents.items():
            if any(kw.lower() in query_lower for kw in config.keywords):
                return config
        for name, pattern in self._keyword_patterns.items():
            if pattern.search(query):
                return self._intents[name]
        return self._intents["casual"]


class ProgressiveDisclosure:
    """渐进披露控制器"""
    
    STAGES = [
        {"name": "summary", "maxItems": 3, "waitFeedback": True},
        {"name": "details", "maxItems": 5, "waitFeedback": True},
        {"name": "context", "maxItems": 8, "waitFeedback": True},
        {"name": "evidence", "maxItems": 12, "waitFeedback": False},
        {"name": "full", "maxItems": 20, "waitFeedback": False},
    ]
    
    def __init__(self):
        self._stage_index = 0
    
    def get_next_stage(self, feedback: Optional[str] = None) -> Dict:
        if feedback == "positive":
            self._stage_index = min(self._stage_index + 2, len(self.STAGES) - 1)
        elif feedback == "negative":
            self._stage_index = max(self._stage_index - 1, 0)
        return self.STAGES[self._stage_index]
```

## 维度三：时序感知召回

### Hindsight 的机制

- `scoreDecayPerDay: 0.02`（固定线性衰减）
- `boostOnUse: 0.1`（使用后提升）
- 简单的 `lastRecalledAt` 记录

### su-memory 的现有能力

su-memory 的 `TemporalSystem` 提供了**分段指数衰减**：

```python
# chrono.py calculate_time_decay()
if days <= 7:
    base_decay = 1.0
elif days <= 30:
    base_decay = 0.95 ** (days - 7)
elif days <= 90:
    base_decay = base_30 * (0.9 ** (days - 30))
else:
    base_decay = base_90 * (0.85 ** (days - 90))

# 干支调制因子
multiplier_map = {"旺": 1.3, "相": 1.15, "休": 1.0, "囚": 0.8, "死": 0.6}
```

### 关键差距

1. su-memory 的衰减是**单向的**（只考虑记忆年龄）
2. Hindsight 有**双向反馈**（使用后 boost，错误后 penalty）
3. 没有与**查询时间**的交互

### 解决方案

**新增文件**: `su_core/_sys/recency_feedback.py`

```python
@dataclass
class FeedbackEvent:
    memory_id: str
    was_useful: bool
    timestamp: int

class RecencyFeedbackSystem:
    """反馈感知的时序系统"""
    
    def __init__(self, temporal_system: TemporalSystem):
        self.temporal = temporal_system
        self._feedback_log: Dict[str, List[FeedbackEvent]] = {}
    
    def record_feedback(self, memory_id: str, was_useful: bool, 
                       timestamp: int = None) -> None:
        ts = timestamp or int(time.time())
        event = FeedbackEvent(memory_id=memory_id, was_useful=was_useful, timestamp=ts)
        self._feedback_log.setdefault(memory_id, []).append(event)
    
    def calculate_recency_score(self, memory_id: str, memory_timestamp: int,
                                memory_wuxing: str, 
                                query_timestamp: int = None) -> float:
        """综合时效性得分"""
        base_decay = self.temporal.calculate_time_decay(memory_timestamp, memory_wuxing)
        recency_boost = self.temporal.calculate_recency_boost(
            memory_timestamp, query_timestamp, memory_wuxing
        )
        feedback_modifier = self._get_feedback_modifier(memory_id)
        return base_decay * recency_boost * feedback_modifier
    
    def _get_feedback_modifier(self, memory_id: str) -> float:
        events = self._feedback_log.get(memory_id, [])
        if not events:
            return 1.0
        recent = events[-3:]
        positive_count = sum(1 for e in recent if e.was_useful)
        if positive_count == len(recent):
            return 1.15
        elif positive_count == 0:
            return 0.85
        return 1.0
```

### Hindsight 的机制

### su-memory 的现有能力

`CausalChain` 提供跨记忆因果关联，但是**静态的**，没有会话级上下文管理。

### 解决方案

```python
@dataclass
class SessionContext:
    session_id: str
    start_time: int
    topics: List[str] = field(default_factory=list)
    memory_ids: List[str] = field(default_factory=list)
    queries: List[str] = field(default_factory=list)

class SessionBridge:
    """多会话上下文桥接器"""
    
    def __init__(self):
        self._sessions: Dict[str, SessionContext] = {}
        self._current_session: Optional[SessionContext] = None
        self._topic_history: Dict[str, List[Dict]] = {}
    
    def start_session(self, session_id: str = None) -> SessionContext:
        sid = session_id or f"session_{int(time.time())}"
        ctx = SessionContext(session_id=sid, start_time=int(time.time()))
        self._sessions[sid] = ctx
        self._current_session = ctx
        return ctx
    
    def calculate_context_boost(self, memory_id: str, 
                                 current_session_id: str = None) -> float:
        ctx = self._current_session or self._sessions.get(current_session_id)
        if not ctx:
            return 1.0
        
        boost = 1.0
        if memory_id in ctx.memory_ids:
            boost *= 1.2  # 当前会话访问过
        
        cutoff = int(time.time()) - 24 * 3600
        recent_visits = sum(
            1 for s in self._sessions.values()
            if s.session_id != ctx.session_id 
            and s.start_time > cutoff 
            and memory_id in s.memory_ids
        )
        if recent_visits > 0:
            boost *= 1.1
        
        return boost
```

**新增文件**: `su_core/_sys/recall_trigger.py`

```python
class RecallTrigger:
    """召回触发器 — 统一的召回入口"""
    
    def __init__(self, 
                 intent_classifier: IntentClassifier = None,
                 session_bridge: SessionBridge = None,
                 wiki_linker: WikiLinker = None,
                 recency_system: RecencyFeedbackSystem = None):
        self.classifier = intent_classifier or IntentClassifier()
        self.session_bridge = session_bridge or SessionBridge()
        self.wiki_linker = wiki_linker or WikiLinker()
        self.recency = recency_system
    
    def should_recall(self, query: str) -> bool:
        """判断是否需要触发召回"""
        intent = self.classifier.classify(query)
        return intent.level >= 1
    
    def recall(self, query: str, 
               memory_store: 'MemoryStore' = None) -> Dict:
        """
        召回主入口
        
        Returns:
        {
            "intent": IntentConfig,
            "dreams_results": [...],
            "wiki_results": [...],
            "context_boost": {...}
        }
        """
        # 1. 意图分类
        intent = self.classifier.classify(query)
        
        # 2. 获取会话上下文
        session_ctx = self.session_bridge.get_session_context()
        
        # 3. 按 level 执行召回
        results = {
            "intent": intent,
            "dreams_results": [],
            "wiki_results": [],
            "context_boost": {}
        }
        
        if intent.level == 0:
            return results
        
        # Dreams 查询
        if intent.level >= 1:
            results["dreams_results"] = self._query_dreams(
                query, intent.tags, session_ctx
            )
        
        # Wiki 查询
        if intent.level >= 2 and intent.wikis:
            results["wiki_results"] = self.wiki_linker.query_wiki(
                query, intent.wikis, intent.tags
            )
        
        # 计算上下文提升
        if session_ctx:
            for mem_id in session_ctx.memory_ids:
                results["context_boost"][mem_id] = \
                    self.session_bridge.calculate_context_boost(mem_id)
        
        # 更新会话记录
        all_ids = [r["id"] for r in results["dreams_results"]] + \
                  [r["name"] for r in results["wiki_results"]]
        self.session_bridge.add_query(query, all_ids)
        
        return results
    
    def _query_dreams(self, query: str, tags: List[str],
                      session_ctx: SessionContext = None) -> List[Dict]:
        """从记忆存储中查询 Dreams"""
        if not memory_store:
            return []
        
        # 基础语义检索
        candidates = memory_store.search(query, top_k=20)
        
        # 应用会话上下文提升
        if session_ctx:
            for cand in candidates:
                if cand["id"] in session_ctx.memory_ids:
                    cand["score"] *= 1.2
        
        # 应用时序反馈
        if self.recency:
            for cand in candidates:
                cand["recency_score"] = self.recency.calculate_recency_score(
                    cand["id"],
                    cand.get("timestamp", 0),
                    cand.get("wuxing", "土")
                )
                cand["score"] *= cand["recency_score"]
        
        candidates.sort(key=lambda x: x.get("score", 0), reverse=True)
        return candidates[:10]
```

## 总结

su-memory 在**易学编码理论**上有独特优势（64卦全息、五行能量、干支时空），但在与外部系统（Wiki）和上层控制流（意图分类、触发器）方面存在明显不足。

本次研究提出的解决方案：
1. **不破坏现有架构** — 所有新增模块都是独立的
2. **渐进增强** — 可以按 phase 逐步实现
3. **充分利用现有优势** — 多跳推理复用 `CausalInference`，时序系统复用 `TemporalSystem`

预期总体提升：**覆盖率 59% → 85%+**
